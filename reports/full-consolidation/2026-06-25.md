## Conflict Policy Applied

When early intermediate branches conflicted with later consolidated branches,
the later consolidated implementation was preserved and the branch ancestry was
recorded with merge commits. This prevented regressions such as:

- Replacing runtime/protocol `runtime.status` and `runtime.shutdown` with older
  C ABI branch code.
- Replacing the data subsystem local-book library/snapshot model with the
  smaller local-content TXT parser crate.
- Replacing the unified Android JNI bridge with duplicate Java/Kotlin
  `com.reader.core.NativeCoreBridge` class definitions.
- Replacing the expanded rule engine with earlier JSONPath slice-only code.

## Android Merge Note

The Android branches had two incompatible definitions of
`com.reader.core.NativeCoreBridge`. The consolidated branch keeps one public
Java `NativeCoreBridge` class as the canonical JNI owner and exposes both:

- queue/poll API used by `ReaderCoreRuntime`
- direct listener API used by the Kotlin sample:
  `pingSmoke`, `runtimeCreate`, `runtimeSend`, `runtimeCancel`,
  `runtimeDestroy`

The duplicate Kotlin `NativeCoreBridge.kt` was removed to avoid a class-name
collision.

## Local Book Merge Note

The data subsystem local-book implementation remains the primary API. The
local-content TXT parser was added as `reader_local_book::txt` and re-exported:

- `parse_txt`
- `parse_txt_with_options`
- `ParsedTxt`
- `TxtChapter`
- `TxtParseOptions`

This preserves the existing `LocalBook`, `LocalBookLibrary`,
`LocalBookLibrarySnapshot`, encoding, and chapter-cache behavior while adding
the standalone TXT parser and edge-case tests.
