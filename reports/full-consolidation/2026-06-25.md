## 冲突处理策略

早期中间分支与后续整合分支冲突时，保留较新的整合实现，并用 merge commit 记录
分支血缘。这避免了以下回退：

- 用旧 C ABI 分支代码覆盖 runtime/protocol 的 `runtime.status` 和
  `runtime.shutdown`。
- 用较小的 local-content TXT parser crate 覆盖 data subsystem 中的 local-book
  library/snapshot model。
- 在 Android JNI 中重新引入重复的 Java/Kotlin `com.reader.core.NativeCoreBridge`。
- 用早期 JSONPath slice-only 代码覆盖已扩展的 rule engine。

## Android 合并说明

Android 分支曾存在两个不兼容的 `com.reader.core.NativeCoreBridge` 定义。合并后
保留一个 public Java `NativeCoreBridge` 作为 JNI owner，并同时暴露：

- `ReaderCoreRuntime` 使用的 queue/poll API
- Kotlin sample 使用的 direct listener API：
  `pingSmoke`、`runtimeCreate`、`runtimeSend`、`runtimeCancel`、`runtimeDestroy`

重复的 Kotlin `NativeCoreBridge.kt` 已移除，避免 class-name collision。

## 本地书合并说明

data subsystem 的 local-book 实现保留为主 API。local-content TXT parser 被添加为
`reader_local_book::txt` 并重新导出：

- `parse_txt`
- `parse_txt_with_options`
- `ParsedTxt`
- `TxtChapter`
- `TxtParseOptions`

这样既保留 `LocalBook`、`LocalBookLibrary`、`LocalBookLibrarySnapshot`、encoding、
chapter-cache 行为，也加入 standalone TXT parser 和 edge-case tests。
